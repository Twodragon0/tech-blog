# PR #396-#403 세션 기록 — L20 real-content + honesty-safe routing 캠페인

## 세션 개요
- 기간: 2026-06-08 ~ 2026-06-10
- 범위: L20 다이제스트 커버가 "파일명 키워드 + TBD KPI" 제네릭 표시에서 **포스트의 실제 보도 내용**(리드 스토리 엔티티 · 출처 · 실제 수집 통계)을 노출하도록 전환하고, 그 과정에서 발생한 아이콘↔헤드라인 불일치를 honesty 게이트를 깨지 않는 방식으로 해결. 이어서 rollup 스펙 정확도 게이트 + 감사 코멘트 정정.
- 결과: 7개 PR 병합 (#396, #397, #398, #399, #401, #402, #403). **#400은 history에 없음(스킵/클로즈)**.
- 핵심 불변식: **honesty 게이트는 BLOCKING**이고 다이제스트는 크론으로 무인 자동발행되므로, 커버가 절대 과대주장(overclaim)하거나 honesty를 FAIL할 수 없어야 한다. 이 캠페인의 모든 설계 판단은 이 제약에서 파생됨.

### PR #398 — feat(cover): content-aware honesty-safe visual routing (fix icon/headline desync)
- 병합: 2026-06-08T18:36:27 +0900 / 커밋 `05bf644a`
- 문제: 밴드 비주얼이 **파일명 키워드**에서 라우팅됐는데(예: "Ransomware" → 자물쇠), 표시 헤드라인은 이제 실제 톱 스토리(예: "Mirai" 봇넷)에서 왔음 → 아이콘이 헤드라인과 모순.
- 해결: 헤드라인이 실제 스토리인 밴드는 **그 스토리에서** 비주얼을 라우팅(`resolve_digest_band_visuals`) → 아이콘이 헤드라인과 일치, 테마 + 히어로 액션도 해소된 비주얼을 따름.
- **construction에 의한 honesty-safe(architect 리뷰)**: 한 줄 헤드라인은 KEYWORD로 라우팅하되, attack-class 빌더는 honesty 게이트가 본문 EVIDENCE TOKEN에 대해 검증하는 특정 사건을 주장한다 — 헤드라인 엔티티만으로는 그 토큰을 보장하지 못함(단일 CVE에서 "CVE REGRESSION CHAIN"으로, 또는 본문 c2/botnet 토큰 없이 "APT36 RAT" → hub_spoke로 라우팅하면 overclaim **AND** 게이트 FAIL). 따라서 모든 attack-class CONTENT 라우팅은 **항상 정직한** `security_advisory`로 다운그레이드. 다이제스트는 무인 크론 발행이므로, 이로써 어떤 커버도 overclaim/honesty-FAIL 불가가 보장됨. 실제 패널이 없는 밴드는 파일명 라우팅 유지.
- 변경: `l20_dispatch.py`의 `resolve_digest_band_visuals`(lockstep 단일 진실원), `_honest_content_visual`(다운그레이드 게이트), `_digest_panels`(공유), `_action_for_visual`. `score_cover_honesty.py`의 `_routed_visual_ids`(path-a)가 동일한 content-aware 라우팅을 replay → scored intent == on-disk visual.
- 재생성: 적격 L20 다이제스트 100개 + 래스터(2026-01-23 .. 2026-06-08).
- 결과: 커버는 neutral/security_advisory 혼합(정직 + 일관). 콘텐츠 라우팅으로 인한 attack-class 모티프가 다이제스트에서 더 이상 나타나지 않음.

## 핵심 학습 (다음 세션을 위해)
1. **honesty는 텍스트가 아니라 비주얼 빌더에 키잉된다.** 표시 텍스트/KPI만 override하면 honesty 클래스가 불변이라 안전(#396). 시각 라우팅을 바꾸는 순간(#398) attack-class overclaim 리스크가 생기므로 `security_advisory`로 강제 다운그레이드해야 함.
2. **다이제스트는 무인 크론 발행 + honesty BLOCKING** → "사람이 검토하니까 괜찮다"가 성립 안 함. construction에 의한 안전성이 필요하고, 그것을 회귀 가드(#399)로 잠가야 한다.
3. **대규모 커버 재생성 PR은 SVG+래스터만, 코드 변경 0**으로 분리(#397). 제너레이터 변경과 corpus 재생성을 같은 PR에 섞지 말 것.
4. **baselined SVG(tests/visual-baselines)와 spec-driven 커버(`_data/*_covers`)는 재생성 대상에서 제외.** spec-driven은 `upgrade_*_cover.py`로만 재생성(L20 제너레이터로 force regen 금지 — CLAUDE.md 경고와 일치).
5. **회귀 가드는 양쪽 taxonomy를 단일 소스에서 파생**시켜 drift 시 자동 FAIL되게 한다(#399의 `CLAIM_CLASSES` 재사용 패턴).
6. Ops Loop의 BLOCKING 체크는 lint-and-types 하나뿐. ruff auto-fix 불가 에러만 커밋하면 됨(#401).

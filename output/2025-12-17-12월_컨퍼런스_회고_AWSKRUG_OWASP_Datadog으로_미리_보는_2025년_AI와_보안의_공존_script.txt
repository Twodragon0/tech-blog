안녕하세요! 클라우드 보안, IT, DevSecOps 전문가로서, 오늘은 2025년 기술 트렌드의 핵심 키워드인 AI와 보안의 공존에 대해 이야기해보려 합니다. 혹시 최근 93%의 보안 리더가 매일 AI 기반 공격을 예상하고 있다는 충격적인 소식을 들으셨나요? 이미지부터 음악, 영상까지 전부 AI로 만드는 시대에, AI가 공격의 도구이자 동시에 방어의 무기가 되고 있다는 사실은 우리에게 시사하는 바가 큽니다. 하지만 우리 같은 클라우드 아키텍트, DevOps 및 보안 엔지니어들에게는 이 화려한 AI 기술 뒤에 숨겨진 Shadow AI, Supply Chain 공격 같은 새로운 보안 위협과 이를 효과적으로 방어할 전략이 더 중요합니다. 오늘은 AWSKRUG AI IDE Kiro, OWASP 서울 챕터, Datadog Security 101 컨퍼런스에서 얻은 인사이트를 바탕으로, AWS re:Invent 2025에서 발표된 Security Agent, GuardDuty Extended, IAM Policy Autopilot 같은 최신 AI 기반 보안 자동화 도구부터, Zero Trust, Post-quantum 암호화, SBOM 구축, 그리고 비용 최적화 전략까지 DevSecOps 관점에서 완벽하게 정리해드리겠습니다. 이 강의가 끝나면 여러분은 변화하는 AI 시대에 맞춰 생성형 AI 프로젝트를 안전하고 효율적으로 운영하는 실질적인 노하우를 갖게 되실 겁니다.

자, 그럼 2025년 보안 트렌드부터 자세히 살펴볼까요? 컨퍼런스들을 관통하는 가장 큰 화두는 역시 AI 보안이었습니다. 앞서 말씀드린 대로, 93%의 보안 리더가 매일 AI 기반 공격을 예상할 정도로 AI는 이제 방어의 도구인 동시에 매우 정교한 공격의 무기가 되고 있습니다.

AI 기반 공격은 어떻게 진화하고 있을까요? 첫째, LLM 기반 피싱입니다. ChatGPT 같은 대규모 언어 모델을 활용해 고도화된 피싱 이메일을 대량으로 생성하는데, 그 자연스러움이 놀랍습니다. 이에 대응하려면 AI 탐지 도구와 사용자 교육을 강화해야 합니다. 둘째, AI 생성 악성 코드입니다. AI가 만든 악성 코드는 탐지 도구를 회피하는 능력이 뛰어나 정적/동적 분석 도구와 행위 기반 탐지를 결합해야 합니다. 셋째, AI 기반 사회공학으로, 개인 정보를 활용한 맞춤형 공격 메시지를 생성해 사용자를 현혹하죠. 다중 인증(MFA)과 Zero Trust 접근 제어가 필수입니다.

특히 주목해야 할 것은 Shadow AI의 위험성입니다. 조직의 승인 없이 사용되는 AI 시스템을 의미하는데요. 민감한 기업 데이터가 승인되지 않은 AI 서비스로 전송되어 데이터 유출 위험이 커지고, GDPR 같은 규정 준수 위반으로 이어질 수 있습니다. 이를 방지하려면 조직 내 AI 사용 정책을 명확히 수립하고 공유해야 합니다. 네트워크 트래픽 모니터링으로 AI 서비스 사용을 탐지하고, 승인된 AI 도구 목록과 사용 가이드를 제공하는 것이 중요합니다. 그리고 정기적인 보안 교육으로 인식도 높여야겠죠.

다음은 Supply Chain 공격의 진화입니다. npm Shai-Hulud 웜처럼 오픈소스 생태계를 노리는 공격이 급증하고 있습니다. 이는 의존성 관리와 SBOM, 즉 소프트웨어 구성 요소 목록 구축의 중요성을 다시 한번 강조합니다. 주요 공격 사례로는 npm Shai-Hulud 웜, 정상 패키지에 악성 코드를 삽입하는 의존성 혼입 공격, 그리고 유사한 이름의 악성 패키지를 배포하는 타이포스쿼팅 등이 있습니다. 이에 대응하려면 SBOM을 구축하여 모든 소프트웨어 구성 요소를 명확히 파악하고, CI/CD 파이프라인에 의존성 스캔을 자동화해야 합니다. 신뢰할 수 있는 소스에서만 패키지를 설치하고, 보안 패치가 포함된 최신 버전으로 정기적인 업데이트를 잊지 말아야 합니다.

Zero Trust Architecture는 이제 업계 표준으로 정착하고 있습니다. ZTNA, 즉 Zero Trust Network Access가 기본이 되면서 "신뢰하지 않고 항상 검증한다"는 원칙이 모든 보안 아키텍처의 핵심이 되고 있죠. 핵심 원칙은 'Never Trust, Always Verify', 'Least Privilege Access', 'Assume Breach', 그리고 'Continuous Monitoring'입니다. 다중 인증과 최소 권한 원칙을 적용하고, 침해를 가정하고 설계하며, 지속적인 모니터링을 통해 위협을 탐지해야 합니다.

마지막으로 Post-quantum 암호화 준비입니다. 양자 컴퓨팅 시대를 대비한 Post-quantum 암호화가 현실화되고 있다는 점인데요. Cloudflare는 이미 전체 트래픽의 52%를 Post-quantum 암호화로 보호하고 있다고 발표했습니다. Google Chrome 브라우저 테스트, AWS KMS 및 CloudHSM 지원, Microsoft Azure Key Vault 지원 등 주요 클라우드 벤더들도 이 흐름에 동참하고 있습니다. 실무에서는 전통 암호화와 Post-quantum 암호화를 동시에 사용하는 하이브리드 방식을 채택하고, 기존 시스템과의 호환성 테스트를 거쳐 중요도가 높은 시스템부터 점진적으로 전환하는 것이 좋습니다.

자, 이제 AWS re:Invent 2025에서 발표된 보안 서비스들을 살펴보겠습니다. AWS는 AI 기반 보안 자동화와 IAM 접근 제어 혁신에 중점을 두었습니다.

먼저 AI 기반 보안 자동화 서비스입니다.
AWS Security Agent는 AI 기반 자동 위협 대응 에이전트로, 실시간 위협 탐지, 사전 정의된 플레이북 기반의 자동 대응, 공격의 전체 맥락을 분석하는 컨텍스트 분석, 그리고 지속적인 학습을 통한 탐지 정확도 향상 기능을 제공합니다. 비정상적인 API 호출이나 네트워크 트래픽을 탐지하고 의심스러운 인스턴스를 격리하거나 보안 그룹 규칙을 자동으로 차단할 수 있습니다.
Security Hub GA의 정식 출시로 멀티 어카운트 보안 통합 관리가 강화되었습니다. 모든 AWS 계정의 보안 상태를 한눈에 확인하는 통합 대시보드를 제공하고, CIS, PCI-DSS 같은 표준 준수 상태를 자동 확인하며, 보안 상태를 점수로 시각화하여 개선 우선순위를 설정할 수 있도록 돕습니다.
GuardDuty Extended Threat Detection은 확장된 위협 탐지 기능으로 더욱 정교한 공격 패턴을 식별합니다. VPC Flow Logs 분석을 통한 네트워크 트래픽 이상 탐지, DNS 로그 분석을 통한 악성 도메인 접근 탐지, CloudTrail 로그 분석을 통한 권한 남용 탐지, 그리고 EKS 보호를 통해 Kubernetes 클러스터의 컨테이너 탈출이나 악성 Pod 탐지까지 가능해졌습니다.

다음은 IAM 및 접근 제어 혁신입니다.
IAM Policy Autopilot은 AI가 최소 권한 원칙에 기반하여 IAM 정책을 자동 생성하고 최적화합니다. 사용자의 실제 사용 패턴을 분석하여 신규 사용자 온보딩 시 자동으로 최소 권한 정책을 생성하고, 기존 정책에서 사용되지 않는 권한을 제거하여 보안을 강화합니다. 또한 필요한 권한을 자동으로 추천하여 개발 생산성을 높이면서도 보안을 유지할 수 있게 돕습니다. 초기 생성된 정책은 반드시 검토 후 적용하고, 프로덕션 환경 적용 전 개발 환경에서 충분히 테스트하는 것이 중요합니다.
AgentCore Identity는 AI 에이전트를 위한 전용 신원 관리 시스템으로, 에이전트 기반 워크로드의 보안을 강화합니다. AI 에이전트에 고유 신원을 부여하고, 세분화된 권한 제어, 모든 행위에 대한 감사 로그, 그리고 안전한 세션 관리를 제공하여 AI 기반 자동화 워크로드의 보안을 한층 더 강화합니다.

이러한 AWS의 서비스들은 AI를 활용한 보안 자동화가 이제 단순한 트렌드를 넘어 실무에서 즉시 활용 가능한 수준에 도달했음을 보여줍니다.

컨퍼런스별 주요 인사이트도 빼놓을 수 없죠.
AWSKRUG AI IDE Kiro Launch Party에서는 AI 기반 개발 생산성 도구인 Kiro IDE가 소개되었습니다. 보안 코드 리뷰와 취약점 자동 탐지 기능이 통합되어 개발 단계에서부터 보안을 고려하는 Shift-Left Security의 실현을 보여주었습니다. AI 기반 코드 리뷰, 자동 보안 스캔, 컨텍스트 인식 제안, AWS 서비스 통합 기능이 핵심입니다. Shift-Left Security를 효과적으로 적용하려면 개발자 교육, CI/CD 파이프라인에 보안 스캔 자동 통합, 보안 체크리스트 활용, 그리고 팀 내 보안 챔피언 지정이 중요합니다.

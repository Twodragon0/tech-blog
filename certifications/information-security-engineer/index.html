다음은 정보보안기사 자격증 준비를 위한 기출문제 및 연습 문제입니다. 각 문제에 답을 선택하고 정답을 확인할 수 있습니다. 각 문제는 관련 포스팅과 연결되어 있어 상세한 설명을 확인할 수 있습니다.

기출문제 리스트

번호 제목 주제 CBT 바로보기 문제다운 해설다운 문제+해설

2024년 기출문제

문제 1 시스템 보안

Q: 운영체제 보안 강화를 위한 조치 중 가장 우선순위가 높은 것은?

정답: A

해설: 보안 패치 적용은 알려진 취약점을 해결하는 가장 기본적이고 중요한 조치입니다. 통합 보안 취약점 점검 및 ISMS-P 인증 대응 실무 포스트에서 자세한 설명을 확인하세요.

문제 2 네트워크 보안

Q: 침입 탐지 시스템(IDS)과 침입 방지 시스템(IPS)의 차이점은?

정답: A

해설: IDS는 침입을 탐지하고 알림만 제공하며, IPS는 탐지 후 자동으로 차단까지 수행합니다. AWS 보안 아키텍처의 핵심 (VPC, GuardDuty - 침입 탐지) 포스트에서 자세한 설명을 확인하세요.

문제 3 암호학

Q: 대칭키 암호화 방식과 비대칭키 암호화 방식의 차이점은?

정답: B

해설: 대칭키 암호화는 동일한 키로 암호화와 복호화를 수행하며, 비대칭키 암호화는 공개키와 개인키 쌍을 사용합니다. AWS 취약점 점검 및 ISMS 대응 가이드 포스트에서 자세한 설명을 확인하세요.

문제 4 정보보안 일반

Q: 정보보안의 3대 요소인 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability) 중 무결성과 관련된 공격은?

정답: C

해설: 무결성은 데이터가 변경되지 않았음을 보장하는 것으로, 데이터 변조 공격이 무결성을 위협합니다. AWS 보안 아키텍처의 핵심 포스트에서 자세한 설명을 확인하세요.

문제 5 정보보안 관리

Q: 접근 통제 모델 중 역할 기반 접근 통제(RBAC)의 특징은?

정답: D

해설: RBAC는 역할에 권한을 부여하고, 사용자에게 역할을 할당하는 방식입니다. AWS 보안 아키텍처의 핵심 포스트에서 자세한 설명을 확인하세요.

2025년 기출문제

문제 6 애플리케이션 보안

Q: SQL Injection 공격을 방어하기 위한 가장 효과적인 방법은?

정답: B

해설: Prepared Statement는 SQL Injection 공격을 방어하는 가장 효과적인 방법입니다. 통합 보안 취약점 점검 및 ISMS-P 인증 대응 실무 포스트에서 자세한 설명을 확인하세요.

문제 7 데이터베이스 보안

Q: 데이터베이스 보안을 강화하기 위한 조치 중 가장 중요한 것은?

정답: A

해설: 데이터베이스 보안은 접근 권한을 최소화하고 데이터를 암호화하는 것이 가장 중요합니다. AWS에서 안전한 데이터베이스 접근 게이트웨이 구축하기 포스트에서 자세한 설명을 확인하세요.

문제 8 무선 네트워크 보안

Q: 무선 네트워크 보안을 강화하기 위한 가장 효과적인 방법은?

정답: C

해설: WPA3 또는 WPA2는 현재 무선 네트워크 보안을 위한 표준 암호화 방식입니다. WEP는 취약하여 사용하지 않아야 합니다. AWS 보안 아키텍처의 핵심 포스트에서 자세한 설명을 확인하세요.

문제 9 전자서명

Q: 전자서명의 주요 목적은?

정답: D

해설: 전자서명은 데이터의 무결성을 보장하고 부인 방지를 제공합니다. AWS 취약점 점검 및 ISMS 대응 가이드 포스트에서 자세한 설명을 확인하세요.

문제 10 사고 대응

Q: 보안 사고 발생 시 첫 번째로 수행해야 할 조치는?

정답: B

해설: 보안 사고 발생 시 먼저 사고를 격리하고 증거를 보존해야 합니다. Datadog SIEM 및 Cloudflare 보안 포스트에서 자세한 설명을 확인하세요.

2026년 기출문제

문제 11 해시 함수

Q: 해시 함수의 주요 특성이 아닌 것은?

정답: A

해설: 해시 함수는 일방향 함수로 역변환이 불가능합니다. AWS 취약점 점검 및 ISMS 대응 가이드 포스트에서 자세한 설명을 확인하세요.

문제 12 방화벽

Q: 방화벽의 기본 정책으로 가장 안전한 것은?

정답: C

해설: 방화벽은 기본적으로 모든 트래픽을 차단하고, 필요한 것만 허용하는 것이 가장 안전합니다. AWS 보안 아키텍처의 핵심 포스트에서 자세한 설명을 확인하세요.

문제 13 취약점 관리

Q: 취약점 관리 프로세스의 올바른 순서는?

정답: D

해설: 취약점 관리는 스캔 → 우선순위 결정 → 패치 적용 → 재검증 순서로 진행됩니다. 통합 보안 취약점 점검 및 ISMS-P 인증 대응 실무 포스트에서 자세한 설명을 확인하세요.

문제 14 보안 정책

Q: 정보보안 정책을 수립할 때 고려해야 할 사항이 아닌 것은?

정답: A

해설: 정보보안 정책은 환경 변화에 따라 정기적으로 검토하고 개선해야 합니다. ISMS-P 인증 완벽 가이드 포스트에서 자세한 설명을 확인하세요.

문제 15 보안 감사

Q: 보안 감사를 수행할 때 가장 중요한 것은?

정답: B

해설: 보안 감사는 객관적이고 체계적으로 수행하고, 발견된 문제에 대해 개선 조치를 취해야 합니다. ISMS-P 보안 감사 포스트에서 자세한 설명을 확인하세요.

문제 16 시스템 보안

Q: 리눅스 시스템에서 SetUID가 설정된 실행 파일의 권한 표기로 올바른 것은?

정답: C

해설: SetUID가 설정되면 소유자의 실행 권한 위치에 's'가 표시됩니다. -rwsr-xr-x는 소유자에게 SetUID가 설정된 상태를 의미합니다. 반면 -rwxr-sr-x는 그룹에 SetGID가 설정된 경우입니다. SetUID 파일은 실행 시 파일 소유자의 권한으로 실행되므로, 보안 점검 시 불필요한 SetUID 파일을 주기적으로 검사해야 합니다.

문제 17 네트워크 보안

Q: iptables 방화벽에서 외부에서 들어오는 SSH(22번 포트) 접속을 차단하는 올바른 명령은?

정답: A

해설: INPUT 체인은 외부에서 들어오는 패킷을 처리하며, -p tcp --dport 22는 TCP 프로토콜의 목적지 포트 22(SSH)를 지정합니다. -j DROP은 해당 패킷을 폐기합니다. SSH는 TCP 프로토콜을 사용하므로 UDP가 아닌 TCP로 지정해야 합니다.

문제 18 암호학

Q: 다음 중 비대칭키(공개키) 암호화 알고리즘이 아닌 것은?

정답: B

해설: AES(Advanced Encryption Standard)는 128/192/256비트 키를 사용하는 대칭키 블록 암호 알고리즘입니다. RSA, ECC(Elliptic Curve Cryptography), ElGamal은 모두 비대칭키(공개키) 암호화 알고리즘입니다. AES는 DES를 대체하기 위해 NIST에서 표준으로 선정한 대칭키 암호입니다.

문제 19 애플리케이션 보안

Q: OWASP Top 10(2021)에서 1위로 선정된 웹 애플리케이션 보안 취약점은?

정답: D

해설: OWASP Top 10 2021 버전에서는 Broken Access Control(취약한 접근 제어)이 1위로 선정되었습니다. 이전 2017 버전에서 1위였던 Injection은 3위로 내려갔습니다. 접근 제어 실패는 인가되지 않은 사용자가 제한된 리소스에 접근할 수 있는 취약점입니다.

문제 20 정보보안 관리

Q: 개인정보보호법에 따라 개인정보처리자가 개인정보 유출 사실을 알게 된 경우, 정보주체에게 통지해야 하는 기한은?

정답: C

해설: 개인정보보호법 제34조에 따르면, 개인정보처리자는 개인정보 유출 사실을 알게 된 때에는 정당한 사유가 없는 한 72시간 이내에 정보주체에게 알려야 합니다. 1,000명 이상의 정보주체에 관한 개인정보가 유출된 경우에는 개인정보보호위원회에도 신고해야 합니다.

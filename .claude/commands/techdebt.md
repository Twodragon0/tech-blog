# 기술 부채 탐지 및 분석

프로젝트의 기술 부채를 탐지하고 우선순위를 제시합니다.

## 탐지 영역

### 1. 중복 코드 패턴
- `_posts/`, `_includes/`, `assets/js/` 디렉토리 스캔
- 동일하거나 유사한 코드 블록 식별
- 3회 이상 반복되는 패턴 추출
- 리팩토링 제안

### 2. 미사용 리소스
- **CSS 클래스**: `assets/css/` 파일에 정의되었으나 사용되지 않는 클래스
- **JavaScript 함수**: `assets/js/` 파일의 호출되지 않는 함수
- **이미지**: `assets/images/`의 참조되지 않는 이미지 파일
- **레이아웃/인클루드**: `_layouts/`, `_includes/`의 사용되지 않는 파일

### 3. TODO/FIXME/HACK 주석
- 모든 파일에서 `TODO`, `FIXME`, `HACK`, `XXX`, `BUG` 주석 검색
- 파일명, 라인 번호, 내용 추출
- 날짜가 오래된 순서로 정렬

### 4. 깨진 참조
- **깨진 링크**: 포스트 내 외부/내부 링크 검증
- **누락 이미지**: Front matter의 이미지 필드가 가리키는 파일 존재 여부
- **누락 인클루드**: `{% include %}` 태그의 파일 존재 여부

### 5. 보안 취약점
- 하드코딩된 API 키, 비밀번호, 토큰 검색
- 의존성 취약점: `npm audit`, `bundle audit` 결과 확인
- CSP 정책 검토 (외부 스크립트 사용 시)

### 6. 성능 이슈
- 최적화되지 않은 이미지 (크기, 포맷)
- 압축되지 않은 CSS/JS 파일
- 지연 로딩이 적용되지 않은 이미지

## 결과 보고 형식

```markdown
## 기술 부채 분석 결과

### 심각도: HIGH
- [ ] 이슈 1: 설명
  - 위치: `파일경로:라인번호`
  - 영향: 보안/성능/유지보수성
  - 제안: 수정 방법

### 심각도: MEDIUM
- [ ] 이슈 2: 설명
  - 위치: `파일경로:라인번호`
  - 영향: 코드 품질
  - 제안: 리팩토링 방법

### 심각도: LOW
- [ ] 이슈 3: 설명
  - 위치: `파일경로`
  - 영향: 가독성
  - 제안: 정리 방법

### 통계
- 총 이슈: N개
- HIGH: X개
- MEDIUM: Y개
- LOW: Z개
```

## 실행 규칙

- **자동 수정 금지**: 사용자가 명시적으로 요청하지 않는 한 절대 자동 수정하지 않음
- **읽기 전용**: 파일을 읽고 분석만 수행
- **우선순위**: HIGH 이슈부터 보고
- **실행 가능 제안**: 구체적인 수정 방법 제시 (코드 예시 포함)

## 명령어

- `/techdebt`: 전체 스캔
- `/techdebt --security`: 보안 취약점만 검사
- `/techdebt --performance`: 성능 이슈만 검사
- `/techdebt --fix`: 자동 수정 가능한 이슈 수정 (명시적 요청 시에만)

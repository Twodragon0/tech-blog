## 2. 2025년 보안 트렌드와 과정의 방향성

### 2.1 왜 지금 클라우드 보안인가?

2025년 보안 환경은 전례 없는 변화를 맞이하고 있습니다:

- **AI 보안의 양면성**: 93%의 보안 리더가 일일 AI 기반 공격을 예상하고 있습니다
- **Shadow AI 위협**: 조직 승인 없이 사용되는 AI 시스템이 새로운 보안 위험으로 부상
- **Supply Chain 공격 급증**: npm Shai-Hulud 웜 등 공급망을 노리는 공격이 정교해지고 있습니다
- **Zero Trust 표준화**: ZTNA가 업계 표준으로 정착하며 "절대 신뢰하지 말고, 항상 검증하라"가 기본 원칙이 되었습니다
- **Post-quantum 암호화**: Cloudflare가 전체 트래픽의 52%를 Post-quantum 암호화로 보호하는 시대

이러한 환경에서 클라우드 시큐리티 8기는 **실무에서 바로 적용 가능한 최신 보안 역량**을 제공합니다.

### 2.4 OT 환경 보안 아키텍처: Purdue 모델

IT-OT 융합 환경에서 Purdue 모델 기반 네트워크 분리 구조:

```
┌─────────────────────────────────────────────────────┐
│ Level 5: Enterprise Network (IT)                    │
│ - ERP, MES, CMMS                                    │
│ - AWS VPC (Production)                              │
│ - Security Hub, GuardDuty                           │
└─────────────────────┬───────────────────────────────┘
                      │ Secure Gateway
                      │ (AWS Transit Gateway + Firewall)
┌─────────────────────┴───────────────────────────────┐
│ Level 4: Site Business Planning & Logistics (IT)    │
│ - Historian, SCADA Server                           │
│ - Read-only access to OT                            │
└─────────────────────┬───────────────────────────────┘
                      │ DMZ (Diode / Unidirectional Gateway)
┌─────────────────────┴───────────────────────────────┐
│ Level 3: Site Operations & Control (OT)             │
│ - HMI, SCADA Clients                                │
│ - ICS Gateway (모든 외부 통신 차단)                 │
└─────────────────────┬───────────────────────────────┘
                      │ ICS Firewall (IEC 62443 준수)
┌─────────────────────┴───────────────────────────────┐
│ Level 2: Supervisory Control (OT)                   │
│ - PLC, DCS, Controllers                             │
│ - Industrial Ethernet (ISO/IEC 11898)               │
└─────────────────────┬───────────────────────────────┘
                      │ Fieldbus
┌─────────────────────┴───────────────────────────────┐
│ Level 1: Basic Control (OT)                         │
│ - Sensors, Actuators                                │
│ - Modbus, OPC-UA                                    │
└─────────────────────────────────────────────────────┘
```

**보안 통제 포인트**:
- **Level 4-5 경계**: AWS Transit Gateway + Network Firewall (Suricata IPS)
- **Level 3-4 경계**: 단방향 게이트웨이 (Data Diode) - OT → IT만 허용
- **Level 2-3 경계**: ICS 전용 방화벽 (Fortinet FortiGate ICS, Claroty)
- **Level 1-2 경계**: Fieldbus 암호화 (TLS 1.3, IEC 62443-3-3 준수)

## 3. 커리큘럼 상세

### 3.1 전체 커리큘럼 개요

클라우드 시큐리티 8기는 총 9주 과정으로 구성됩니다:

| 주차 | 주제 | 핵심 내용 |
|------|------|----------|
| 1주차 | 인프라의 본질부터 보안의 미래까지 | 네트워크, 서버, 가상화 기초, 2025 보안 트렌드(AI 보안, Zero Trust) |

| 5주차 | AWS Control Tower 및 보안 도구 | Landing Zone, SCP, Security Hub, Datadog SIEM, Cloudflare |
| 6주차 이후 | DevSecOps 심화 | CI/CD 보안, 컨테이너 보안, AgentCore Identity, 통합 정리 |

### 3.2 주요 학습 영역

#### DevSecOps 영역

- **보안 통합 개발**: CI/CD 파이프라인에 보안 자동화 통합
- **컨테이너 보안**: Docker, Kubernetes 보안 모범 사례
- **코드 보안**: GitHub Advanced Security, 정적/동적 분석
- **인프라 보안**: IaC 보안, Terraform 보안 모범 사례

#### FinOps 영역

- **비용 최적화**: AWS Cost Explorer, 예약 인스턴스, Savings Plans
- **리소스 태깅 전략**: 비용 할당 및 추적
- **거버넌스**: AWS Control Tower, SCP를 통한 거버넌스
- **예산 관리**: 예산 알림 및 비용 이상 탐지

#### 클라우드 보안 영역

- **네트워크 보안**: VPC 설계, Security Group, NACL
- **접근 제어**: IAM 정책, 최소 권한 원칙
- **위협 탐지**: GuardDuty, CloudTrail, Config
- **데이터 보호**: S3 암호화, KMS 활용

### 3.3 실습 환경 구성 예시

과정에서 다루는 AWS 보안 환경 구성의 기본 예시입니다:

```bash
# AWS CLI를 사용한 보안 환경 기본 구성

# 1. VPC 생성 (보안을 위한 네트워크 분리)
aws ec2 create-vpc --cidr-block 10.0.0.0/16 \
  --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=SecureVPC},{Key=Environment,Value=Training}]'

# 2. CloudTrail 활성화 (감사 로그)
aws cloudtrail create-trail --name security-audit-trail \
  --s3-bucket-name my-cloudtrail-logs \
  --is-multi-region-trail \
  --enable-log-file-validation

# 3. GuardDuty 활성화 (위협 탐지)
aws guardduty create-detector --enable \
```

```python
# AWS Cost Explorer API를 사용한 월별 비용 분석
# 파일: cost_analysis.py

import boto3
from datetime import datetime, timedelta
import json

def analyze_monthly_cost():
    ce = boto3.client('ce', region_name='us-east-1')

    # 지난 3개월 비용 조회
    end_date = datetime.now().date()
    start_date = end_date - timedelta(days=90)

    response = ce.get_cost_and_usage(
        TimePeriod={
            'Start': start_date.strftime('%Y-%m-%d'),
            'End': end_date.strftime('%Y-%m-%d')
        },
        Granularity='MONTHLY',
        Metrics=['UnblendedCost'],
        GroupBy=[
            {'Type': 'DIMENSION', 'Key': 'SERVICE'},
            {'Type': 'TAG', 'Key': 'Environment'}
        ]
    )

    # 비용 상위 5개 서비스 출력
    print("=== 월별 상위 비용 서비스 ===")
    for result in response['ResultsByTime']:
        period = result['TimePeriod']['Start']
        print(f"\n[{period}]")

        groups = sorted(result['Groups'],
                       key=lambda x: float(x['Metrics']['UnblendedCost']['Amount']),
                       reverse=True)[:5]

        for group in groups:
            service = group['Keys'][0]
            cost = float(group['Metrics']['UnblendedCost']['Amount'])
            print(f"  {service}: ${cost:.2f}")

if __name__ == '__main__':
    analyze_monthly_cost()
```

```yaml
  push:

        with:
          scan-type: 'fs'
          scan-ref: '.'
          format: 'sarif'
          output: 'trivy-results.sarif'

      # 4. Secret Scanning
      - name: Gitleaks
        uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: {% raw %}${{ secrets.GITHUB_TOKEN }}{% endraw %}

      # 5. Docker 이미지 빌드
      - name: Build Docker image
        run: docker build -t myapp:{% raw %}${{ github.sha }}{% endraw %} .

      # 6. Container Image Scanning
      - name: Scan Docker image with Trivy
        run: |
          trivy image --severity HIGH,CRITICAL --exit-code 1 myapp:{% raw %}${{ github.sha }}{% endraw %}

      # 7. Image Signing (Sigstore Cosign)
      - name: Install Cosign
        uses: sigstore/cosign-installer@v3

      - name: Sign container image
        run: |
          cosign sign --key cosign.key myapp:{% raw %}${{ github.sha }}{% endraw %}
        env:
          COSIGN_PASSWORD: {% raw %}${{ secrets.COSIGN_PASSWORD }}{% endraw %}

      # 8. SARIF 결과 업로드 (GitHub Security Tab)
      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        if: always()
        with:
          sarif_file: 'trivy-results.sarif'

  deploy:
    needs: security-scan
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
```

## 7. Threat Hunting Queries

### 7.1 Splunk SPL 쿼리 (보안 위협 탐지)

```spl
<!-- SIEM Detection Query 1: IAM 권한 상승 탐지 -->
index=aws sourcetype=aws:cloudtrail eventName IN (
    "AttachUserPolicy",
    "AttachGroupPolicy",
    "AttachRolePolicy",
    "PutUserPolicy",
    "PutGroupPolicy",
    "PutRolePolicy"
)
| where userIdentity.principalId!="AIDAI*" AND userIdentity.principalId!="AROAI*"
| eval policy_arn=coalesce(requestParameters.policyArn, "InlinePolicy")
| search policy_arn IN ("*AdministratorAccess*", "*PowerUserAccess*", "*FullAccess*")
| stats count by userIdentity.userName, eventName, policy_arn, sourceIPAddress
| where count > 3
| eval severity="HIGH"
| table _time, userIdentity.userName, eventName, policy_arn, sourceIPAddress, severity

<!-- SIEM Detection Query 2: 의심스러운 EC2 인스턴스 생성 (암호화폐 채굴 탐지) -->
index=aws sourcetype=aws:cloudtrail eventName=RunInstances
| spath input=requestParameters path=instanceType output=instance_type
| where instance_type IN ("p3.16xlarge", "p3dn.24xlarge", "p4d.24xlarge", "g4dn.16xlarge")
| eval cost_per_hour=case(
    instance_type=="p3.16xlarge", 24.48,
    instance_type=="p3dn.24xlarge", 31.22,
    instance_type=="p4d.24xlarge", 32.77,
    instance_type=="g4dn.16xlarge", 4.35
)
| stats sum(cost_per_hour) as total_cost by userIdentity.userName, sourceIPAddress
| where total_cost > 100
| eval alert="Potential Cryptomining Activity"
| table _time, userIdentity.userName, sourceIPAddress, total_cost, alert

<!-- SIEM Detection Query 3: S3 데이터 유출 탐지 -->
index=aws sourcetype=aws:cloudtrail eventName IN (
    "GetObject",
    "ListObjects",
    "ListBucket"
)
| stats count, sum(eval(if(responseElements.totalSize!=null, responseElements.totalSize, 0))) as total_bytes
  by userIdentity.userName, requestParameters.bucketName, sourceIPAddress
| where total_bytes > 10737418240
| eval total_gb=round(total_bytes/1073741824, 2)
| eval severity=case(
    total_gb>100, "CRITICAL",
    total_gb>50, "HIGH",
    total_gb>10, "MEDIUM"
)
| table _time, userIdentity.userName, requestParameters.bucketName, sourceIPAddress, total_gb, severity

<!-- SIEM Detection Query 4: 비인가 리전 리소스 생성 탐지 -->
index=aws sourcetype=aws:cloudtrail
| rex field=awsRegion "(?<region>.*)"
| where region NOT IN ("ap-northeast-2", "us-east-1", "ap-southeast-1")
| stats count by userIdentity.userName, eventName, region, sourceIPAddress
| where count > 5
| eval alert="Unauthorized Region Activity"
| table _time, userIdentity.userName, eventName, region, sourceIPAddress, alert

<!-- SIEM Detection Query 5: GuardDuty 고위험 탐지 알림 -->
index=aws sourcetype=aws:guardduty severity>=7
| spath input=service.action path=actionType output=action_type
| stats count by title, type, severity, resource.instanceDetails.instanceId, action_type
| eval risk_level=case(
    severity>=8, "CRITICAL",
    severity>=7, "HIGH"
)
| table _time, title, type, severity, risk_level, resource.instanceDetails.instanceId, action_type
```

### 7.2 Azure Sentinel KQL 쿼리

```kql
// Query 1: AWS CloudTrail 이상 행위 탐지 (Sentinel에서 AWS 로그 수집 시)
AWSCloudTrail
| where TimeGenerated > ago(24h)
| where EventName in (
    "DeleteBucket",
    "DeleteDBInstance",
    "DeleteTrail",
    "StopLogging",
    "DeleteFlowLogs"
)
| extend UserName = tostring(UserIdentityPrincipalid)
| extend SourceIP = tostring(SourceIpAddress)
| summarize Count=count() by UserName, EventName, SourceIP, bin(TimeGenerated, 1h)
| where Count > 3
| project TimeGenerated, UserName, EventName, SourceIP, Count
| order by TimeGenerated desc

// Query 2: 비정상 로그인 시도 (MFA 실패)
SigninLogs
| where TimeGenerated > ago(7d)
| where ResultType != 0
| where AuthenticationRequirement == "multiFactorAuthentication"
| summarize FailedAttempts=count() by UserPrincipalName, IPAddress, bin(TimeGenerated, 1h)
| where FailedAttempts > 5
| project TimeGenerated, UserPrincipalName, IPAddress, FailedAttempts
| order by FailedAttempts desc

// Query 3: 의심스러운 파일 다운로드 (대용량)
OfficeActivity
| where TimeGenerated > ago(24h)
| where Operation == "FileDownloaded"
| extend SizeInMB = todouble(Size) / 1048576
| where SizeInMB > 100
| summarize TotalDownloadMB=sum(SizeInMB) by UserId, ClientIP, bin(TimeGenerated, 1h)
| where TotalDownloadMB > 500
| project TimeGenerated, UserId, ClientIP, TotalDownloadMB
| order by TotalDownloadMB desc
```

## 5. 마무리

클라우드 시큐리티 8기는 단순한 교육 과정이 아닙니다. **대체 불가능한 클라우드 보안 전문가로 성장하는 여정**의 시작점입니다.

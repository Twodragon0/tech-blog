## Fixes

### Korean separator residue in SVG text

`f742e1f4` ‚Äî **fix(svg):** collapse Korean separator residue after Hangul strip

`_strip_hangul()` was stripping Hangul codepoints but leaving adjacent separator
characters (U+00B7 middle-dot `¬∑`, U+2022 bullet `‚Ä¢`). Fixed by normalising the
string after the Hangul pass: whitespace collapsed, leading/trailing separators
removed.

`c4fca4b2` ‚Äî **fix(svg-titles):** regen 4 digest covers to clear middle-dot residue

Four covers already in the repository contained `¬∑` in their `<title>` elements.
Regenerated from spec to produce clean ASCII-only titles, satisfying the gate added
in `64823037`.

### Cross-platform case-sensitivity (spec slugs)

`ebff34a1` ‚Äî **fix(spec-case):** align Amazon_Q + AWSin spec slugs with post `image:` refs

Two spec YAML files had slugs that differed in case from the post `image:` field ‚Äî
invisible on macOS but a broken reference on Linux CI. Corrected to match the
canonical post filename casing.

### Pre-commit shell path safety

`8a5aa1e6` ‚Äî **fix(pre-commit):** use array+printf for SVG paths to handle whitespace safely

The pre-commit hook was iterating SVG paths via word-splitting. Any path containing a
space would split into two tokens and cause a false positive or miss. Replaced with a
`mapfile`/array approach using `printf '%s\0'` and `while IFS= read -r -d ''`.

### Post validation

`3d6d7119` ‚Äî **fix(post):** 2026-05-27 digest validation cleanup

Front-matter and excerpt hygiene for the digest post published today. Removes
trailing whitespace and normalises the `excerpt` field that caused the pre-commit
post-validator to warn.

### Cover size gate

`8707b887` ‚Äî **fix(covers):** trim 2025-05-09 ISMS cover to fit HQ size band

The ISMS cover rendered by `d2ac7c8a` measured above the 73728-byte HQ cap enforced
by `check_svg_size_gate.py`. Trimmed redundant nodes to bring file size within band.

## Migration notes for contributors

- **macOS clone:** run `git config core.ignorecase false` in this clone to surface
  case-drift locally before pushing to CI. See
  `docs/troubleshooting/CROSS_PLATFORM_CASE_SENSITIVITY.md` for the full workflow.
- **Pre-commit hook:** regenerated automatically by `scripts/install-hooks.sh`.
  Clones updated since this release pick it up via
  `git config core.hooksPath .githooks`. If your hook is older than `8a5aa1e6`
  (2026-05-27), re-run `scripts/install-hooks.sh` to get the shell-safe path fix.
- **Spec authoring:** slug values in `assets/images/digest_covers/*.yml` must match
  the post `image:` field exactly (case-sensitive). The new gate (`79fab399`) will
  block commits where they differ.
